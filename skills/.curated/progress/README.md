# progress

`progress` runs a strict production-progress protocol for existing products.

Use it when Codex should inspect an internal app, find the best real improvement, coordinate with other active worktrees, get one approval, create a fresh worktree, execute through a dependency graph with safe worker waves, QA the result, and package formal plus plain-English demo summaries.

It creates the same artifact set every time under `plans/<slug>/`, including discovery, approval, shared contract, execution graph, plain-English before/after summaries, verification, demo, final delivery, and handoff files. It also maintains `plans/progress-registry.md` so parallel runs can avoid duplicating or conflicting with active work.

Install:

```bash
$skill-installer install https://github.com/vanderhaka/skills/tree/main/skills/.curated/progress
```

Restart Codex after installing new skills.
