# progress

`progress` runs a strict production-progress protocol for existing products.

Use it when Codex should inspect an internal app, find the best real improvement, get one approval, create a fresh worktree, execute through a dependency graph with safe worker waves, QA the result, and package a short demo.

It creates the same artifact set every time under `plans/<slug>/`, including discovery, approval, shared contract, execution graph, verification, demo, final delivery, and handoff files.

Install:

```bash
$skill-installer install https://github.com/vanderhaka/skills/tree/main/skills/.curated/progress
```

Restart Codex after installing new skills.
